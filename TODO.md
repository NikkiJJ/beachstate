# BeachState To-Do

Internal checklist for product and legal-risk safeguards.

## Information-Only Safeguards

- [ ] Use clear informational wording across UI copy (not guidance language).
- [ ] Add visible disclaimer in onboarding, key data views, and Terms pages.
- [ ] Avoid binary safety labels like "safe" or "unsafe".
- [ ] Show data source and "last updated" timestamp for every surfaced data point.
- [ ] Add terms covering no warranty, limitation of liability, and user responsibility.
- [ ] Keep logs of source timestamps/responses used to render user-facing results.
- [ ] Show explicit "unavailable" or "stale" states for missing/outdated data.
- [ ] Avoid claims such as "guaranteed" or "real-time" unless contractually true.
- [ ] Display geographic coverage caveats clearly (for example, England-only sources).
- [ ] Schedule periodic legal review of copy, disclaimer text, and Terms.

## Product UX Checks

- [ ] Add "For information only. Not safety guidance." notice in app UI.
- [ ] Add "Check official local advice" link near each location result.
- [ ] Keep provider attribution and data freshness visible on result cards.
